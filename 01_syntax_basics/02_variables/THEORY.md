# Переменные

## Что такое переменная?
Переменная это коробка, которая хранит значение. Имеет тип **Название = значение**

**Название** переменной это то, как к ней можно _обращаться_. Её **значение** это то, что она _хранит_ в себе.

**"="** это знак _присваивания_, а не равно как в математике. (знак сравнения же обозначается как "==").

```python
user_age = 5 # присваивание значения
minimal_age = 10 # присваивание значения

user_last_name = None # Это если переменная нужна, но ещё нет значения, то можно использовать и такое =)

print(user_age >= minimal_age) # --> False, "==" оператор сравнения
```

Python имеет **динамическую типизацию**, а это значит что одна переменная может хранить в себе _любые данные_: строки, числа, функции, объекты и многое другое.
```python
value = 5 # это число
print(type(value)) # --> <class 'int'>
 
value = "hello" # а это строка
print(type(value)) # --> <class 'str'>
```


Переменные чувствительные к регистру.

### Примеры
```python
name = "bob" # Переменная с именем "name" имеет значение "bob"(строка)
Name = "Alan" # Это другая переменная не имеющая ничего общего с "name"
age = 4 # переменная с именем "age" имеет значение "4"(число)
user = get_current_user() # переменная с именем "user" имеет значение "get_current_user()"(функция)
data = {"age": 25} # переменная с именем "data" имеет значение "{"age": 25}"(словарь)
fruits = ["apple", "banana"] # переменная с именем "fruits" имеет значение "["apple", "banana"]"(список)

# это просто примеры, подробнее о типах данных будет сказано в следующем модуле.
```

## Правила именования
Переменные нельзя называть ключевыми словами,
начинать имя переменной с цифры, специальных символов,
называть можно с заглавной/маленькой буквы и нижнего подчеркивания,
если переменная имеет название из двух слов, то стоит использовать snake_case,
константы называются в верхнем регистре.

### Плохие имена
```python
for = 4 # Нельзя, так как "for" ключевое слово(в будущем поймете почему)
1name = "Vasiliy" # Нельзя, ибо начинается с цифры
%age = 9 # Нельзя, так как начинается со специального символа
```

### Хорошие имена
```python
_age = 4 # Можно, всё хорошо
Age = 4 # Тоже можно
age = 3 # Тоже можно
user_age = 10 # snake_case для переменных из более, чем одного слова
number_of_staff = 99 # тоже snake_case

PI = 3.1415 # константа, пишется в верхнем регистре
API_KEY = "тут ключ от апишки, но его нельзя хранить так открыто(на будущее)"
```

**Старайтесь давать осмысленные имена переменным!**

### Неправильно
```python
x = "Adam"
y = 6
w = "Peterson"

xs = x + " " + w + " " + str(y) + " years old"

print(xs) # --> Adam Peterson 6 years old

# Но не понятные имена переменных, что неприемлемо!

```

### Правильно
```python
user_first_name = "Adam"
user_last_name = "Peterson"
user_age = 6

user_info = user_first_name + " " + user_last_name + " " + str(user_age) + " years old"
print(user_info) # --> Adam Peterson 6 years old

# Но теперь всё значительно понятнее!
```

## Интересная фича

```python
user_name, user_age, user_country = "Adam", 17, "US"

```

## Краткая шпаргалка

- `name = value` — создание переменной
- `=` — присваивание, `==` — сравнение
- Имена: `snake_case`, константы: `UPPER_CASE`
- Тип определяется автоматически (динамическая типизация)
- `None` — специальное значение "ничего"
