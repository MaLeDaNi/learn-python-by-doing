## Комментарии

Комментарии нужны чтобы, как ни странно, комментировать что-то.

Они помогают программистам лучше понять структуру кода, его логику, взаимодействие элементов, а так же места, где надо что-то доработать.
При выполнении программы компьютер игнорирует комментарии.


### Существует два вида комментариев

**Однострочные**

Начинаются с символа **"#"** и их не надо закрывать. Используются для пояснения каких-то частей кода или чтобы временно убрать какую-то строку кода и найти ошибку.
```python
# Это однострочный комментарий
x = 10 # И это тоже, но уже после кода
```

**Многострочные(docstring)**

Начинаются с тройных кавычек и закрываются также **("""текст""")**. Они используются для документирования чего-либо. Идут сразу после объявления функции, класса, метода или модуля(пока для тебя это неизвестные слова, но потом они обретут смысл).
```python
"""
это многострочный комментарий
"""

def say_hello():
    """
    Это многострочный docstring
    Эта функция выводит "Hello World" в консоль
    (или какая-то более сложная логика и её документация тут)
    """
    print("Hello World")

def say_goodbye():
    """Это однострочный docstring"""
    print("Goodbye")
```

### Как НЕ надо писать комментарии
```python
# Присваиваем значение переменной x равное 5
x = 5

# Так не надо делать!
# Не нужно дублировать информацию из кода
```

```python
# Произошли изменения и переменная x теперь хранит значение 10

# Присваиваем значение переменной x равное 5
x = 10
# Что? где 5?

# Нужно поддерживать свои комментарии
# Иначе может случиться как тут и в итоге возникнет путаница с тем что комментарий уже не соответствует действительности
```
